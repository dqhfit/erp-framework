1Tổng quan hệ thống

Migration MSSQL chuyển đổi từng module nghiệp vụ (sales, inventory, HR…) từ ứng dụng legacy SQL Server sang ERP Framework. Mỗi module được xử lý độc lập — không cần chuyển tất cả cùng lúc.

Thành phầnVai trò
settings/migrationGiao diện điều khiển toàn bộ quy trình
migration-router80+ tRPC endpoints quản lý manifest + job
migration-workerpg-boss worker xử lý job nền
migration-full-importEngine streaming ETL với auto-resume
mssql-connectionsCRUD kết nối MSSQL (mật khẩu AES-256-GCM)
migration-plan/modules/*.yamlSingle source of truth mỗi module (Git-tracked)

Luồng dữ liệu

MSSQL Legacy                    ERP Framework
──────────────────              ─────────────────────────────
Tables          ──ETL──────▶    entity_records (PostgreSQL)
Stored Procs    ──AI codegen──▶ Procedures JS/TS (Tier B/D)
Enum Tables     ──materialize─▶ Bảng enums
Schema/FK       ──codegen──────▶ Entity definitions, Page mẫu

Pipeline tổng thể

Discover→ Diagram→ Enrich→ Capture→ Generate→ Data→ Review→ Audit / Cutover

2Yêu cầu trước khi bắt đầu

Phía server ERP

  • Server đang chạy (pnpm dev hoặc production build)
  • ENCRYPTION_KEY đã đặt (mã hóa mật khẩu MSSQL)
  • Biến môi trường LLM: ANTHROPIC_API_KEY hoặc LLM profile trong Settings → AI
  • Thư mục migration-plan/modules/ tồn tại ở gốc repo

Phía MSSQL Legacy

  • Tài khoản DB read-only (SELECT trên tất cả bảng cần migrate)
  • Nếu cần Capture Golden: quyền EXECUTE stored proc, bật allowWrite trên kết nối (tạm thời)
  • Kết nối từ server ERP tới SQL Server phải thông (test qua tab Connections)
  • SQL Server 2012+ (cần sys.dm_exec_procedure_stats)

Quyền trong ERP

ℹ️

Cần role admin trên công ty. Tất cả migration endpoint yêu cầu rbacProcedure("edit", "settings").

4Quy trình 8 bước

Vào Settings → Migrations. Thanh bên trái liệt kê module đã tạo. Mỗi module đi qua 8 tab từ trái sang phải.

① Discover — Khám phá

Tự động quét MSSQL, xây dựng sơ đồ bảng + stored proc cho module.

1
Nhấn "Tạo module mới"
  • Tên module: chữ thường, không dấu, không khoảng trắng (vd: sales)
  • Seed tables: 1–3 bảng trung tâm (vd: dbo.Sales_Order,dbo.Sales_Item)
  • Exclude tables: bảng log, temp, archive cũ
  • Max tables: giới hạn tổng (mặc định 30 — tăng cho module lớn)
2
Nhấn "Chạy Discover"

Job chạy nền. Theo dõi tiến trình qua thanh trạng thái (WebSocket real-time).

3
Kết quả

File migration-plan/modules/<module>.yaml được tạo với danh sách bảng + cột + FK + stored proc.

⚠️

BFS đi theo FK — nếu bỏ sót bảng quan trọng, thêm vào Seed Tables rồi chạy lại. Bảng không liên quan dùng "Loại trừ bảng" trong UI để gỡ bỏ sau.

② Diagram — Sơ đồ

Visualize quan hệ giữa các bảng, xác nhận scope module.

  • Node xanh = entity, node vàng = enum
  • Edge = FK relation; cross-module edge được tô màu riêng
  • Click node → xem cột + FK chi tiết

Kiểm tra cần làm:

  • Node "lạ" không thuộc module → thêm vào Exclude
  • Bảng danh sách cố định (trạng thái, loại, loại hình) → đổi Kind thành enum

③ Enrich — Làm giàu

AI đặt tên tiếng Việt, mô tả, gợi ý tier cho từng bảng/stored proc.

1
Enrich tự động (batch)

Nhấn "Enrich toàn bộ". Cấu hình: maxCostUsd (giới hạn chi phí LLM, mặc định $5), skipEnriched (bỏ qua đã xử lý — an toàn chạy lại).

2
Chỉnh sửa thủ công
  • Đổi tên entity → cascade tự động sang FK relations
  • Đổi kind entity ↔ enum → cascade entityType trên field FK
  • Đổi tên field → cập nhật mapTo.field
3
Normalize Names (AI)

AI đề xuất đổi tên hàng loạt cho nhất quán (vd: orderId, orderNo, madon → đồng nhất thành orderId).

ℹ️

Mọi thay đổi ghi vào decisions.yaml và được tái sử dụng cho module sau.

④ Capture Golden — Chụp chuẩn vàng

Chạy stored proc trên MSSQL thật, lưu output làm baseline kiểm tra sau migration.

⚠️

Yêu cầu kết nối MSSQL bật allowWrite = true (tạm thời). Tắt lại ngay sau khi capture xong.

1
Sinh test cases (AI)

Chọn stored proc → nhấn "Sinh test cases (AI)" → AI tạo 10 case (happy path / boundary / edge). Xem preview và sửa tham số nếu AI sinh sai.

2
Capture

Nhấn "Capture" → thực thi proc trên MSSQL → lưu output vào e2e/golden/<module>/<proc>.json.

3
Kiểm tra kết quả

Xem số case thành công / thất bại. Case thất bại = proc cần quyền đặc biệt hoặc dữ liệu không tồn tại.

⑤ Generate — Sinh code

AI dịch T-SQL stored proc → JavaScript/TypeScript tương đương.

TierLoại procOutputVị trí
B Đọc dữ liệu + logic đơn giản JS function Bảng procedures
D Logic phức tạp, external call TS plugin packages/plugins/module-<name>/
A C Quá phức tạp / domain cụ thể Stub + ghi chú Developer tự viết tay
1
Dry-run trước

Chọn proc → "Xem trước (Dry-run)" → AI sinh code, hiển thị preview. KHÔNG lưu.

2
Xem T-SQL gốc

Nếu code sinh sai, nhấn "Xem T-SQL" → so sánh logic gốc → sửa trong preview editor.

3
Áp dụng

Nhấn "Áp dụng": Tier B → upsert bảng procedures; Tier D → ghi file .ts vào packages/plugins/.

⑥ Data — Import dữ liệu

ETL dữ liệu từ bảng MSSQL vào entity_records PostgreSQL.

🚨

Đây là bước di chuyển dữ liệu thật. Thực hiện trên staging trước, xác nhận ổn rồi mới chạy production.

Cách 1 — Quick Migrate (bảng nhỏ, đồng bộ)

Tab Data → chọn bảng → chọn Limit (để test với 100 rows) → nhấn "Migrate".

Cách 2 — Full Import Job (dataset lớn, có auto-resume)

Nhấn "Full Import" trên thanh trên → chọn bảng + cấu hình → nhấn "Bắt đầu Import". Xem chi tiết tại Mục 6: Full Import.

⑧ Audit + Cutover — Kiểm toán & Chuyển giao

1
Chạy Audit (Dry-run)

AI đọc manifest + T-SQL + code sinh + golden baselines → sinh markdown checklist: rủi ro, điểm cần kiểm tra, khuyến nghị.

2
Lưu báo cáo

Xem + chỉnh → "Lưu báo cáo" → migration-plan/audit/<module>.md

3
Finalize module

Đảm bảo Review đạt ngưỡng + E2E golden tests pass → nhấn "Finalize module" → phase chuyển sang live, ghi cutoverAt.

4
Thao tác MSSQL

Đổi tên proc cũ: EXEC sp_rename 'dbo.sp_GetOrder', 'dbo.sp_GetOrder_DEPRECATED'

Theo dõi error log 7 ngày. Nếu có call đến proc cũ → chưa migrate đủ.

5Phân loại Stored Procedure (Tier)

TierMô tảKhi nào dùng
A Developer viết từ đầu Logic quá phức tạp, AI không tạo được chính xác
B AI sinh → bảng procedures (JS) Đọc dữ liệu, filter, join, tính toán cơ bản. Chỉnh sửa không cần deploy.
C Developer viết, tham khảo AI Logic trung bình, AI sinh đúng ~70% nhưng cần chỉnh tay
D AI sinh → TypeScript plugin file Gọi external API, xử lý file, workflow phức tạp. Cần deploy sau khi thêm file.

Quy tắc chọn tier nhanh

Đặc điểm procTier
Chỉ SELECT + JOIN + WHEREB
Có INSERT/UPDATE nhưng logic đơn giảnB
Gọi stored proc khác (nested exec)B hoặc D
Xử lý cursor, loop phức tạpD
Gửi email, gọi HTTP, đọc fileD
Transaction phức tạp + rollbackD hoặc A

6Full Import — Import toàn bộ dữ liệu

Cơ chế ETL chính cho dataset lớn (hàng triệu rows), với auto-resume sau server crash và streaming theo PK-order.

Luồng hoạt động

Tạo job → phát hiện PK từ MSSQL → ghi migrationFullJobTables
         ↓
Worker nhận job → với mỗi bảng (status != done):
         ↓
    streamReadByPk(lastPk, batchSize=1000)
         ↓ mỗi batch:
    UPSERT entity_records (theo PK — không duplicate khi resume)
    Cập nhật lastPk + rowsImported
    Phát WebSocket { kind: "full-progress" }
         ↓
allDone → completed  |  someDone → paused  |  else → failed

Auto-resume sau server restart

✅

Khi server khởi động lại, hệ thống tự tìm jobs có status running/queued/paused và re-enqueue. Worker tiếp tục từ lastPk đã lưu — không bị duplicate, không mất dữ liệu.

Theo dõi tiến trình

Vào Settings → Migrations → Jobs import:

StatusÝ nghĩaHành động
runningĐang xử lýTheo dõi progress bar
pausedTạm dừng hoặc server restartNhấn "Tiếp tục"
completedXong toàn bộ bảngKiểm tra rowsImported
failedTất cả bảng đều lỗiXem log lỗi, tạo lại

Bảng không có PK đơn

  • Không có PK: thêm cột id IDENTITY vào MSSQL trước khi import
  • Composite PK: dùng Quick Migrate (tab Data) với limit nhỏ
  • PK kiểu varchar: hỗ trợ — streaming theo thứ tự lexicographic

Pause / Resume / Cancel

  • Pause: nhấn "Dừng" → status = paused
  • Resume: nhấn "Tiếp tục" → re-enqueue, tiếp từ checkpoint
  • Cancel: nhấn "Hủy" → status = canceled (không thể resume, phải tạo lại)

7Xử lý sự cố thường gặp

Lỗi kết nối MSSQL

Triệu chứng: "Connect ECONNREFUSED" hoặc "Login failed"

1
Kiểm tra firewall

Test-NetConnection -ComputerName <host> -Port 1433

2
SQL Server đang lắng nghe?

netstat -an | findstr 1433 trên máy SQL Server

3
Trust Server Cert

Môi trường dev: bật. Production với CA hợp lệ: tắt.

Full Import bị treo (heartbeat không cập nhật > 5 phút)

Status = running nhưng không có progress mới.

  • Query MSSQL bị lock: EXEC sp_who2 → tìm blocking sessions → kill nếu cần
  • Network timeout: giảm batchSize xuống 500
  • Server restart: restart ERP server → auto-resume ngay khi boot

Nếu không tự resume: nhấn "Dừng" → "Tiếp tục" để force re-queue.

Discover quét quá nhiều bảng không liên quan

  • Giảm maxTables (vd: 20)
  • Thêm bảng cầu nối không liên quan vào excludeTables
  • Chọn seed tables chính xác hơn (bảng TRUNG TÂM, không phải bảng lookup)
  • Sau discover: dùng "Loại trừ bảng" trong UI để gỡ bảng thừa

Enrich AI sinh nhãn sai hoặc tier sai

  • Sửa label/description thủ công trong tab Enrich
  • Đổi tier: chọn proc → dropdown "Đổi tier"
  • Enrich lại từng proc: "Enrich riêng proc này" → dry-run → apply

Generate sinh code JS lỗi logic

  • Xem T-SQL gốc (nút "Xem T-SQL") → so sánh với code sinh ra
  • Dry-run lại → sửa trong preview editor → Apply
  • Đổi sang Tier D hoặc Tier A nếu quá phức tạp
  • Golden test sẽ bắt lỗi logic: pnpm e2e:full --module <module>

White screen / reload khi dùng migration UI

✅

Đã sửa trong vite.config.ts với watch.ignored: ["**/migration-plan/**"]. Cần restart Vite dev server (Ctrl+C → pnpm dev) để có hiệu lực.

8Vận hành sau Cutover

Monitoring 7 ngày đầu

1
SQL Server error log

Kiểm tra có call đến proc _DEPRECATED không → nếu có, còn code chưa chuyển.

2
ERP application log

Lọc level=error liên quan đến module mới.

3
User feedback

Thu thập báo cáo anomaly từ người dùng portal/admin.

Rollback nếu cần

1
Unfinalize module

Nút "Hủy finalize" trong tab Audit → phase về filled

2
Khôi phục proc cũ trên MSSQL

Đổi tên _DEPRECATED về tên gốc

3
Điều tra + sửa → Finalize lại

Sau khi sửa code gen + golden test pass → Finalize lần 2

Dọn dẹp sau ≥ 30 ngày ổn định

  • Xóa proc _DEPRECATED trên MSSQL (backup DB trước)
  • Tắt kết nối MSSQL (Settings → Connections → Delete) — giữ 1 kết nối read-only nếu cần tra cứu
  • Thư mục migration-plan/ giữ trong Git (audit trail)

Thêm module mới sau khi đã live

Hệ thống hỗ trợ thêm module bất kỳ lúc nào — quay lại Bước 1 Discover. Quyết định từ module cũ (decisions.yaml) tự động gợi ý cho module mới.
